# Design Knowledge Source Package 001

## Propósito

Este paquete convierte cuatro fuentes fundamentales de diseño en conocimiento documental trazable para fortalecer criterio, gusto, razonamiento centrado en las personas y capacidad de evaluación. No pretende imitar una estética fija ni convertir preferencias personales en reglas universales.

El objetivo operativo es que el sistema actúe como una extensión analítica superior de Jonathan Martínez:

- preservando intención, restricciones y decisiones;
- elevando claridad, coherencia, detalle y calidad visual;
- distinguiendo gusto, evidencia, estándar, hipótesis y decisión;
- detectando contradicciones antes de producir;
- proponiendo una dirección fuerte en lugar de combinaciones arbitrarias;
- sometiendo el juicio experto a validación cuando la consecuencia lo requiera.

## Fuentes

1. Don Norman — *The Design of Everyday Things*.
2. Steve Krug — *Don't Make Me Think, Revisited*.
3. Jakob Nielsen y Kara Pernice — *Eyetracking Web Usability*.
4. Sophie Lovell — *Dieter Rams: As Little Design as Possible*.

## Regla de síntesis

El criterio combinado se expresa así:

> Reducir la complejidad accidental sin borrar señales; construir jerarquía para el escaneo; diseñar feedback y recuperación; mantener coherencia sistémica; y comprobar el juicio contra uso real, accesibilidad y contexto.

## Límites

- Los PDFs no forman parte del paquete.
- No se almacena texto completo ni reproducción extensa.
- Los principios son derivados y redactados originalmente.
- Ningún principio sustituye estándares técnicos ni pruebas.
- Rams se utiliza como filosofía y referencia visual, no como norma universal.
- Eye tracking produce hipótesis sobre atención, no prueba automática de comprensión.
- La documentación no activa RAG, runtime, skills ni cambios de producto.
