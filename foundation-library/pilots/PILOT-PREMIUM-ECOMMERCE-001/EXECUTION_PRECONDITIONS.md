# Precondiciones de ejecución

Estado: `DEFINED_NOT_EXECUTED`.

Codex solo podrá construir o probar este piloto mediante una autorización posterior que declare repositorio aislado, HEAD, rama, rutas, código permitido, comandos, navegador, evidencia, rollback y criterios de terminación.

La ejecución deberá usar exclusivamente fixtures sintéticas, sin credenciales ni proveedores de pago. El repositorio no podrá ser un producto real ni Symphonie. Toda dependencia deberá declararse y justificarse. Las pruebas deberán cubrir los ocho anchos canónicos, teclado, foco, semántica, contraste, movimiento reducido, contenido extremo y estados del carrito.

Un resultado visual satisfactorio no equivale a validación de backend, integración o producción. Cada oráculo debe registrar `PASS`, `FAIL` o `NOT_EXECUTED`; nunca se infiere aprobación por ausencia de evidencia.
